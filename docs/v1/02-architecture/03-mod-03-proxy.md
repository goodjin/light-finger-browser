### 1.2 核心职责

- 从池中选择可用代理
- 将代理与实例建立一对一绑定
- 回收、解绑与健康检查
- 通过 Provider 适配器刷新代理池

## 3. 接口定义

| 接口编号 | 方法 | 说明 |
|---------|------|------|
| API-M3-001 | Acquire(country, type) | 获取最优可用代理 |
| API-M3-002 | Release(proxyID) | 释放代理 |
| API-M3-003 | Bind(proxyID, instanceID) | 绑定代理到实例 |
| API-M3-004 | Unbind(proxyID) | 解绑代理 |
| API-M3-005 | HealthCheck(proxyID) | 健康检查 |
| API-M3-006 | RefreshPool() | 从 Provider 刷新代理池 |

### 3.1 核心数据结构

```go
type Proxy struct {
    ID          string
    IP          string
    Port        int
    Country     string
    Type        ProxyType
    Status      ProxyStatus
    BindID      string
    SuccessRate float64
    Latency     int
    Provider    string
}
```

---

## 4. 业务规则

1. 已绑定代理不得绑定到第二个实例。
2. 代理选择优先按成功率降序、延迟升序。
3. 健康检查失败时需衰减成功率，并在阈值下将状态置为 `dead`。
4. 实例销毁后应释放或解绑代理，避免悬挂绑定。
