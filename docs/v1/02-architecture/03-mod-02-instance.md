## 1. 系统定位

### 1.1 在整体架构中的位置

```
┌─────────────────────────────────────────────────────┐
│      Wails 应用层 / 诊断采样 / 上层编排调用              │
└───────────────────────┬─────────────────────────────┘
                        │ ▼ 调用
┌─────────────────────────────────────────────────────┐
│               ★ MOD-02 实例管理模块 ★                │
│   端口分配 / 进程生命周期 / CDP 接入 / 持久化状态      │
└───────────────────────┬─────────────────────────────┘
                        │ ▼ 依赖
┌─────────────────────────────────────────────────────┐
│   MOD-01 指纹引擎 / 浏览器运行时发现 / 代理绑定信息      │
└─────────────────────────────────────────────────────┘
```

### 1.2 核心职责

- 创建、销毁、查询浏览器实例
- 维护实例状态与端口分配
- 建立并缓存 CDP 连接
- 将浏览器运行时路径解析与实例启动流程串联

### 3.1 状态机

```
PENDING -> STARTING -> RUNNING -> STOPPING -> STOPPED
                  \-> ERROR
```

### 3.2 核心数据结构

```go
type BrowserInstance struct {
    ID          string
    Status      InstanceStatus
    Fingerprint *fingerprint.Fingerprint
    ProxyID     string
    AccountID   string
    CDPEndpoint string
    PID         int
    Port        int
    UserDataDir string
}
```

---

## 4. 设计约束

1. 单实例必须绑定独立端口与用户数据目录。
2. 运行时路径优先级遵循浏览器产物发现契约：环境变量覆盖 > `artifacts.json` > 打包相对路径。
3. CDP 连接仅对 `RUNNING` 实例开放。
4. 实例模块负责生命周期编排，不负责定义指纹字段本身。
