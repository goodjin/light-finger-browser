## 1. 当前目标

当前活动文档集只服务一个目标：**把现有 Go/Wails 编排层与未来自建 Chromium-based stealth 浏览器内核计划对齐**。

当前仓库聚焦两层：

1. **已实现的运行时编排层**：指纹、实例、代理、浏览器运行时发现
2. **待执行的浏览器内核层**：源码基线、patch overlay、构建、签名、发布、验证

## 4.1 当前执行焦点

当前浏览器内核专项不再从“泛泛的下一阶段”表述推进，而是以已落库的 ready 任务集为起点：

| Ready 任务 | 目标 | 说明 |
|-----------|------|------|
| `BROW-001` | browser-core 工作区规范 | 先固化独立工作区、目录边界和版本文件 |
| `BROW-002` | 构建机准入要求 | 先固化 CPU / RAM / disk / OS / toolchain 基线 |
| `BROW-006` | 浏览器产物输出布局 | 先定义目录结构、频道、命名与产物入口 |
| `BROW-014` | 检测基线与发布门禁规范 | 先固化采样字段、检测站与 release gate |

> `BROW-003`、`BROW-004`、`BROW-005`、`BROW-007` 及后续源码拉取/编译任务，均应在上述 ready 任务产出后再进入执行。

## 7. 风险与依赖

| 风险/依赖 | 影响 | 缓解措施 |
|----------|------|---------|
| 缺少真实 Chromium 工作区 | 无法进入浏览器源码构建 | 将 browser-core 作为独立工作区准备 |
| 构建机磁盘/算力不足 | 阻塞全平台产物构建 | 准备专用构建环境 |
| 当前运行时仍保留 CloakBrowser 命名 | 文档与心智可能继续偏向竞品 | 逐步以“browser runtime / self-built browser”口径替换 |
| 诊断覆盖未完全门禁化 | 难以在发版前量化 drift | 将采样与差异结果接入 release gate |
