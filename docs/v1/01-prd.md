## 1. 项目概述

### 1.1 背景

当前仓库的目标已经收敛为：**构建一个自主可控的指纹浏览器产品，以及承接该产品的 Go/Wails 运行时编排层**。

当前已经具备的基础：

1. 指纹模型与一致性校验
2. 实例生命周期与 CDP 接入
3. 代理绑定与健康检查
4. 面向打包产物的浏览器运行时发现

当前仍需补齐的核心能力：

1. 自维护 Chromium 基线
2. stealth patch overlay
3. 全平台构建、签名、发布与回滚链路
4. 运行时实际指纹与期望指纹的验证闭环

### 1.2 产品目标

1. 以 **Chromium 源码** 为基线，自主构建 macOS / Windows / Linux 浏览器产物。
2. 完整掌控浏览器内核、补丁、指纹注入和版本演进节奏。
3. 让当前仓库承担 **指纹、实例、代理、运行时发现、诊断验证** 这层职责。
4. 在可控性、可验证性、交付能力上，做出一款整体上超过 CloakBrowser 的产品。

### 1.3 非目标

以下内容不再属于当前 PRD 范围：

1. TikTok/Facebook 等具体业务站点自动化
2. 账号注册、接码、内容生成、批量发布等上层业务系统
3. 与浏览器产品无直接关系的业务运营流程设计

## 4. 用户故事

### US-001: 创建一个可复现的隔离浏览器实例

作为浏览器运行时开发者，我想基于固定指纹、固定代理和确定的浏览器版本创建实例，以便稳定复现问题并运行自动化流程。

**验收标准**：

- AC-001-01：实例创建时能绑定独立 user-data-dir、端口、代理
- AC-001-02：相同 seed 生成的期望指纹稳定可复现
- AC-001-03：实例启动后可通过 CDP 建立连接
- AC-001-04：实例异常退出不会污染其他实例状态

### US-002: 交付一套可被应用层消费的浏览器产物

作为浏览器内核开发者，我想产出带 manifest 和版本信息的跨平台浏览器包，以便桌面端和服务端都能消费同一套产物契约。

**验收标准**：

- AC-002-01：每个平台产物都能通过 manifest 找到唯一入口
- AC-002-02：manifest 记录版本、平台、架构、路径和校验信息
- AC-002-03：应用层无需改启动逻辑即可切换到新版本产物

### US-003: 在发布前验证真实运行时指纹

作为 QA / Release 工程师，我想采集浏览器实际运行时指纹并与期望值对比，以便在发版前识别 drift 和未覆盖字段。

**验收标准**：

- AC-003-01：可采集 UA、platform、locale、timezone、screen、WebGL、canvas、audio 等关键字段
- AC-003-02：差异结果可追踪到实例和时间点
- AC-003-03：发布前能明确区分“已覆盖字段”“未覆盖字段”“已偏移字段”

## 5. 核心数据实体

| 编号 | 实体名称 | 描述 |
|------|---------|------|
| Entity-001 | BrowserInstance | 浏览器实例的运行状态、端口、用户目录、CDP 入口与关联资源 |
| Entity-002 | FingerprintProfile | 指纹种子与生成后的期望环境参数 |
| Entity-003 | ProxyEndpoint | 代理地址、类型、绑定状态、健康状态 |
| Entity-004 | BrowserArtifactManifest | 多平台浏览器产物入口、版本和校验元数据 |
| Entity-005 | FingerprintSnapshot | 实际运行时采集到的浏览器指纹快照 |

## 6. 业务规则

| 编号 | 规则名称 | 规则内容 | 优先级 |
|------|---------|---------|-------|
| Rule-001 | 一实例一代理 | 实例运行期间代理保持稳定，释放或销毁后才可解绑回收 | P0 |
| Rule-002 | 指纹一致性 | platform、timezone、locale、screen、GPU 等字段必须满足一致性约束 | P0 |
| Rule-003 | 浏览器版本锁定 | 同一实例必须显式关联一个浏览器版本，不允许隐式漂移 | P0 |
| Rule-004 | 运行时发现优先级 | 路径解析顺序固定为 env override > manifest > app-relative bundled path | P0 |
| Rule-005 | 发布门禁 | 未通过指纹验证、启动冒烟或签名校验的产物不得进入 stable | P1 |

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 跨平台 | 至少支持 macOS、Windows、Linux 主流桌面环境 |
| 可维护性 | 运行时编排层与浏览器内核工作区边界清晰，不在当前仓库直接 vendoring 全量 Chromium |
| 可验证性 | 指纹采样与差异报告可重复执行，结果可追踪 |
| 可回滚性 | 至少保留最近两个稳定浏览器版本，并支持 manifest 级版本切换 |
| 开发可调试性 | 保留 `CLOAKBROWSER_PATH` / `BROWSER_BINARY` 作为开发覆盖入口，但不作为生产唯一依赖；该命名仅表示兼容期遗留入口 |
